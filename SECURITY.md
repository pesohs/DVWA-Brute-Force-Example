# Security Policy

## Supported Versions

This project is intended **strictly for educational and authorized security testing purposes only**.  
Because of the nature of the project, security updates may not follow a fixed schedule.

| Version | Supported          |
|--------|----------------------|
| Latest | :white_check_mark:   |
| Older  | :x:                  |

If you are using an older version, updating to the latest commit is strongly recommended.

---

## Responsible Disclosure

If you discover any vulnerabilities **within this repository’s code**, please report them responsibly.

### How to Report a Vulnerability

1. Github: @0xpesohs
2. Use the subject line: **"[SECURITY] Vulnerability Report"**
3. Include:
   - A clear description of the issue  
   - Steps to reproduce  
   - Potential impact  
   - Suggested fixes (optional but appreciated)

Please **do NOT** open public issues for security-related reports.  
This prevents exploitation and protects users.

---

## Scope & Ethical Use

This project is provided **for learning, research, and testing in controlled environments only**, such as:

- Systems you personally own  
- Systems you have explicit written permission to test  
- Labs (TryHackMe, HTB, local VMs, demo servers)

**Unauthorized use of this tool against live, public, or private systems that you do not own or control is illegal and prohibited.**

The author is **not responsible for misuse** of the code.

---

## Reporting Misuse

If you believe someone is using your fork or a derivative of this project for malicious or unauthorized activity, please notify:

- GitHub Support  
- Your local authorities or CERT organizations  

Include any relevant evidence so the situation can be addressed quickly.

---

## Security Best Practices

If you're experimenting with this script:

- Always run in isolated test environments  
- Never store credentials in plain text  
- Avoid pushing real passwords or access logs to the repository  
- Rotate any credentials used for testing  
- Consider using rate‑limiting, IP blocking, or logs when testing on your own systems
